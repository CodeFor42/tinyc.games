Features

  • Cross-platform, working on Win, Mac, Linux, ...
  • Needs no configuration nor setup
  • Just click to compile and run
  • As little as 200 LOC for each game
  • Built on SDL (Simple DirectMedia Layer)
  • Runs with TCC (the Tiny C Compiler)
  • REAL ACTUAL GAMES

What's wrong?!

It is way too hard to get C code running these days if you aren't already set up for it. An interested user could get lost in hours/days/weeks of setup and configuration before ever getting something to run. It's very discouraging.

Very sad!

Let's fix it!

The goal

The goal of tinyc.games is to have something if you're interested in C game programming (the only RIGHT way to do it... right?!), such that:

  • You can click a thing to run real code in less than a minute after downloading it.
  • You can look at the code and understand it sometime soon (~200 lines of simple code).
  • You can makes changes to a game and run it with your changes.
  • NO additional configuration, downloading or installing required.

How close are we to the goal?

Pretty close? We've included everything you need on Windows. Linux requires you to install a couple standard things from your package manager. And on Mac... well... anyone know how to get TCC to link with SDL on Mac?

Further testing on more machines always helps!
